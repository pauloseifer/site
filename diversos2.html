1.11. Git

  • Inicialização de novo repositório

https://github.com/pauloseifer/memoria https://gist.github.com/xirixiz/b6b0c6f4917ce17a90e00f9b60566278

Depois de criar o repositório no github, no diretório em que vai ficar o origin

git init git add README.md (preciso?) git branch -M main (lembrar que isto substitui o "master") git remote add origin git@github.com:pauloseifer/NOVOREPOSITORIO.git (na página está errado) git add -A (para levar tudo) git commit -m "comentário" git push –set-upstream origin main (primeiro push - se escrever errado, no "erro" vem o comando certo) git push (para os demais)

  • word/docx

Criar arquivo .gitattributes com conteúdo *.docx diff=pandoc

  • git ftp (https://github.com/git-ftp/git-ftp)

Configurar:

git config git-ftp.url "ftp://ftp.example.net/public_html" git config git-ftp.user "ftp-user" git config git-ftp.password "secr3t"

Depois git ftp init git ftp push

Para baixar do repositório de outro lugar configura como nas linhas acima git ftp download (não faz merge)
