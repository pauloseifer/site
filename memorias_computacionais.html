1.1. MySQL

  • Senha root (logado como root): mysqladmin -u root password senha
  • no mysql: CREATE USER 'pseifer'@'localhost' IDENTIFIED BY 'senha';
  • todos privilégios: GRANT ALL PRIVILEGES ON . TO 'pseifer'@'localhost';
  • Para acesso externo:
    • https://phoenixnap.com/kb/mysql-remote-connection
    • CREATE USER 'myuser'@'localhost' IDENTIFIED BY 'mypass';
    • CREATE USER 'myuser'@'%' IDENTIFIED BY 'mypass';
    • GRANT ALL ON . TO 'myuser'@'localhost';
    • GRANT ALL ON . TO 'myuser'@'%';
    • FLUSH PRIVILEGES;

  • E-mail / Gmail linha de comando

Install it with the following commands:

sudo apt-get update
sudo apt-get install ssmtp
Edit /etc/ssmtp/ssmtp.conf to look like this:

root=rpi3abc@gmail.com
mailhub=smtp.gmail.com:465
FromLineOverride=YES
AuthUser=rpi3abc@gmail.com
AuthPass=testing123
UseTLS=YES
Send a one-liner like so:

echo "Testing…1…2…3" | ssmtp myusername@gmail.com
or

printf "Subject: Test\n\nTesting…1…2…3" | ssmtp myusername@gmail.com

(interactive)
(split-window-right (floor (* 0.65 (window-width))))
(other-window 1)
(switch-to-buffer "org.org")
(split-window-below (floor (* 0.68 (window-height))))
(other-window 2)
(split-window-below (floor (* 0.8 (window-height))))
(other-window 0)
(sql-mysql) ;; usa outra janela aberta como alvo do mysql
)

4. Git

  • Inicialização de novo repositório

https://github.com/pauloseifer/memoria
https://gist.github.com/xirixiz/b6b0c6f4917ce17a90e00f9b60566278

Depois de criar o repositório no github, no diretório em que vai ficar o origin

git init
git add README.md (preciso?)
git branch -M main (lembrar que isto substitui o "master")
git remote add origin git@github.com:pauloseifer/NOVOREPOSITORIO.git (na página está errado)
git add -A (para levar tudo)
git commit -m "comentário"
git push –set-upstream origin main (primeiro push - se escrever errado, no "erro" vem o comando certo)
git push (para os demais)

  • word/docx

Criar arquivo .gitattributes com conteúdo
*.docx diff=pandoc

  • git ftp (https://github.com/git-ftp/git-ftp)

Configurar:

git config git-ftp.url "ftp://ftp.example.net/public_html"
git config git-ftp.user "ftp-user"
git config git-ftp.password "secr3t"

Depois
git ftp init
git ftp push

Para baixar do repositório de outro lugar
configura como nas linhas acima
git ftp download
(não faz merge)

5. Criptografia

  • Teste com gpg ok. O gpg mantém a senha em uma "memória" por alguns minutos. Conveniente caso se esqueça ela, ou registre errado.
